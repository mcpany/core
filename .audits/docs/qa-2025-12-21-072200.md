## 3. Gap Analysis & Remediation
**Context**: The `product-evolution-plan.md` and `docs/roadmap.md` identified "Admin Management API (CRUD)" as a High Priority / In Progress feature.

**Finding**:
*   `server/pkg/admin` previously implemented only Read-Only operations (`ListServices`, `GetService`, etc.) and `ClearCache`.
*   It lacked `CreateService`, `UpdateService`, and `DeleteService`.

**Remediation Actions Taken**:
1.  **Protocol Update**: Modified `proto/admin/v1/admin.proto` to define `CreateService`, `UpdateService`, and `DeleteService` RPCs.
2.  **Code Implementation**: Updated `server/pkg/admin/server.go` to implement these methods by leveraging the `ServiceRegistryInterface`.
3.  **Dependency Wiring**: Updated `server/pkg/app/server.go` to inject the `ServiceRegistry` into the `AdminServer`.
4.  **Testing**: Verified changes with unit tests in `server/pkg/admin`.
